**3. safe_exploration_stages.svg — ❌ 수정 필요**

**SVG 코드 검증:**
- **화살표 정의 누락**: X축(`line x1="80" ... x2="520"`)에서 `marker-end="url(#arrow)"`를 참조하고 있으나, `<defs>` 섹션에는 `id="arrowBlue"`와 `id="arrowOrange"`만 정의되어 있음. `id="arrow"`가 없어 브라우저에서 X축 화살표가 렌더링되지 않음.
- **좌표 불일치 (타원 크기)**:
  - X축 스케일: (520-80)px / (27.5-22.5)unit = 440px / 5unit = 88 px/unit (아님, Ticks 간격으로 재계산 필요).
  - Ticks 간격 확인: 110(22.5) ~ 205(23.75). 차이 95px. 값 차이 1.25. 95/1.25 = **76 px/unit**.
  - **1단계 원(±1%, 0.5 unit)**: 0.5 * 76 = 38px (지름). 반지름(rx)은 19px이어야 함. 코드: `rx="24"`. (26% 오차)
  - **2단계 원(±2%, 1.0 unit)**: 1.0 * 76 = 76px (지름). 반지름(rx)은 38px이어야 함. 코드: `rx="48"`. (26% 오차)
  - 3단계 원(±5%, 2.5 unit): 2.5 * 76 = 190px (지름). 반지름(rx)은 95px이어야 함. 코드: `rx="95"`. 정확함.
- **화살표 연결**:
  - 1->2단계 화살표(`id="arrowBlue"` 사용): 끝점 y=233. 2단계 타원 상단 y = 300 - 72(ry) = 228. 화살표가 타원 내부 5px 지점에서 끝남.
  - 2->3단계 화살표(`id="arrowOrange"` 사용): 끝점 y=165. 3단계 타원 상단 y = 300 - 140(ry) = 160. 화살표가 타원 내부 5px 지점에서 끝남.
  - 화살표 끝이 타원 경계에 정확히 닿지 않음.

**판정 사유:**
SVG 파일에서 치명적인 화살표 정의 누락 오류와 타원 크기(좌표) 불일치, 화살표 연결 좌표 오차가 발견됨. PNG 2장은 완벽하나 SVG 오류로 인해 챕터 전체 수정 필요.
